## 🔧 模块职责说明

### 1. **models.py** - 数据模型层
- **Config**: 系统配置类，管理所有 API 和运行时参数
- **TaskContext**: 任务上下文，封装单个翻译任务的所有信息
- **StageResult**: 阶段执行结果，用于各处理阶段之间的数据传递
- **FinalResult**: 最终翻译结果，包含完整的翻译信息和状态

### 2. **logging_config.py** - 日志基础设施
- **ColorFormatter**: 彩色控制台日志格式化器
- **GUILogHandler**: GUI 文本控件日志处理器
- **setup_logger()**: 统一的日志初始化函数

### 3. **config.py** - 配置管理层
- 默认提示词模板（Draft/Review）
- 目标语言列表常量
- GUI 配置参数

### 4. **prompt_builder.py** - 提示词构建
- **PromptBuilder.build_user_message()**: 根据任务和阶段动态构建用户消息
- 支持 Draft 和 Review 两个阶段
- 自动集成 TM 建议

### 5. **fuzzy_matcher.py** - 模糊匹配引擎
- **FuzzyMatcher.find_best_match()**: 使用 thefuzz 库进行字符串相似度匹配
- 支持多进程加速（大数据量时）
- 可配置的匹配置信度阈值

### 6. **concurrency_controller.py** - 并发控制
- **AdaptiveConcurrencyController**: 自适应并发度控制器
- 根据请求成功率动态调整并发数
- 实现冷却机制和成功累积策略

### 7. **terminology_manager.py** - 术语库管理
- **TerminologyManager**: 术语库管理器
- 异步加载、查询、更新术语库
- 后台异步写入机制
- 支持 Excel 格式的导入导出

### 8. **api_stages.py** - API 处理阶段
- **APIDraftStage**: 初译阶段，调用 API 生成初始翻译
- **APIReviewStage**: 校对阶段，优化初译结果
- **LocalHitStage**: 本地术语命中检查
- 统一的 API 调用接口和重试机制

### 9. **workflow_orchestrator.py** - 工作流编排
- **WorkflowOrchestrator**: 工作流编排器
- 协调各个处理阶段的执行顺序
- 处理新旧文档的不同翻译模式
- 统一的结果构建和错误处理

### 10. **gui_app.py** - GUI 应用层
- **TranslationApp**: 主应用程序界面
- 文件选择、语言配置、提示词编辑
- 任务执行控制和进度显示
- 异步任务调度和日志集成

## 🎯 解耦优势

### 1. **单一职责原则**
每个模块只负责一个明确的业务领域，代码更易理解和维护。

### 2. **依赖关系清晰**
模块间通过明确定义的接口通信，降低耦合度。

    ↓
任务处理流程:
1. TerminologyManager.find_similar() → TM 建议

   ├─ LocalHitStage (本地命中)
