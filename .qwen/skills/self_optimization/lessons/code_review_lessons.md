**版本**: 1.0  
**创建日期**: 2026-04-03

## 🔴 关键教训

### 1. 必须分层审查

**问题**: 初次审查未能发现12对重复文件

**教训**:
- ✅ 审查必须按层进行：架构→模块→代码→风格
- ✅ 每层有独立的检查清单
- ✅ 不能跳跃审查

**复用场景**: 所有代码审查任务

---

### 2. 自动化工具是基础

**问题**: 手动扫描效率低且易遗漏

**教训**:
- ✅ 优先使用自动化工具（radon, mypy, import-linter）
- ✅ 工具无法解决时再手动检查
- ✅ 所有质量指标必须基于工具输出

**复用场景**: 所有代码审查任务

---

### 3. 量化数据不可少

**问题**: 使用了"约15%"等模糊表述

**教训**:
- ✅ 所有指标必须精确
- ✅ 必须使用工具生成数据
- ✅ 不能使用估算或猜测

**复用场景**: 所有报告生成任务

---

### 4. 修复必须验证

**问题**: P0/P1修复后未立即运行验证

**教训**:
- ✅ 每次修复后必须运行编译检查
- ✅ 必须运行相关测试
- ✅ 必须验证功能完整性

**复用场景**: 所有Bug修复任务

---

## 📝 审查清单

### 架构层
- [ ] 检查分层架构是否遵守
- [ ] 检查依赖方向是否正确
- [ ] 检查循环导入
- [ ] 检查模块职责是否清晰

### 模块层
- [ ] 检查重复文件
- [ ] 检查`__init__.py`导出
- [ ] 检查导入路径规范
- [ ] 检查公共接口完整性

### 代码层
- [ ] 检查代码重复率
- [ ] 检查函数复杂度
- [ ] 检查类大小
- [ ] 检查异常处理

### 风格层
- [ ] 检查PEP 8规范
- [ ] 检查类型注解
- [ ] 检查文档字符串
- [ ] 检查命名规范

## 🛠️ 推荐工具

### 重复代码检测
```bash
# 使用radon
radon cc . -a -s

# 使用pylint
pylint --disable=all --enable=similarities .
```

### 架构依赖验证
```bash
# 使用import-linter
import-linter --config .import-linter
```

### 复杂度分析
```bash
# 使用radon
radon mi . -s

# 使用mypy
mypy --strict .
```
